Virtual Smart Tutor

Create a digital learning tool to help children learn the basics of a subject. The system must work without a parent or guardian actively guiding the learning. An parent or guardian should be able to open the app and leave it with a child with no worries.

Specifics: Any approved platform is allowed. Additionally, this is the task that would likely benefit the most from being a virtual reality application.

Multi-User Whiteboard

Create a digital whiteboard that can be synced across multiple internet-connected clients, to provide an easy way for students and teachers to connect. The app should be accessible from at least two different devices across a local network and sync with as minimal delay as possible.

Specifics: Web applications are preferred and are likely easier to create an app for. However any approved platform is allowed.

Assignment Tracker

Create a digital assignment tracker with a place to input an overview, details, due dates, or set reminders. A required feature for this specific project is an assignment reminder system, that hooks into the OS-level APIs to deliver alerts, for example, as a push notification to a users smart phone or tablet.

Specifics: Building the assignment tracker as a native mobile app is highly recommended as alert integration at the OS-level is easier and in the real world, people would likely benefit from this app being on their phone. However any approved platform is allowed.

Submissions can only be written as one of the following:
  • Web Application

    Any web application tech stack is permitted (e.g. MEAN, MERN). Any frontend framework is permitted, but Angular or ReactJS is recommended for developers familiar with one of those frameworks. This is not at all a requirement and will not count against or towards any submissions.

  • Native Mobile Application

    Mobile apps written for the iOS or Android mobile operating systems are permitted. We recommend that iOS developers use Swift and Android developers use Java or Kotlin.

  • Native Desktop Application

    Desktop applications written for Windows, MacOS, or Linux are permitted. All submissions must be compiled for the x86 or x64 binaries. Unfortunately, we cannot accept apps compiled for ARM at this time. All desktop frameworks are permitted (e.g. WPF, UWP, Swift, Qt, Electron JS).

  • Virtual Reality Application

    Virtual reality applications submitted must be compatible with SteamVR and SteamVR headsets and run on Windows. 64-bit (x64) applications are preferred. 32-bit (x86) applications are permitted. Unfortunately, we cannot accept apps compiled for ARM at this time.
